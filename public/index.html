75XP Web API Reference

All request and response bodies are JSON. Protected endpoints require a valid access token in the Authorization: Bearer <token> header. Tokens are issued on login and registration, and can be refreshed via /api/auth/refresh without re-authentication.

For the frontend team

Topic Detail
Access token Short lived JWT. Returned in the response body. Include as Authorization: Bearer <token> on all protected requests.
Refresh token Longer lived token. Set and rotated as an HttpOnly cookie refreshToken. Not accessible via JavaScript. Rotated on every successful refresh.
Session invalidation Changing a password PATCH /api/profile/password invalidates all existing sessions. The user must log in again. No other endpoint currently triggers this.
Token expiry When the access token expires, the client should call POST /api/auth/refresh using the cookie. If the refresh token is also expired, the user must log in again from scratch.
Login error Login returns the same 401 and same error message for both a wrong password and an unknown email. I have done this on purpose.

Authentication

Access token is returned in the response body and should be included in the Authorization header for protected endpoints.

POST /api/auth/register

Creates a new user account. On success, returns the new user's details along with an access token. A refresh token cookie is also set.

Request Body
Field Type Required Notes
email string Yes Must be a valid email address.
name string Yes Minimum 3 characters.
password string Yes See validation rules
avatar_url string Yes Must be a fully qualified URL of an image 
{
  "email": "testuser@example.com",
  "name": "Test User",
  "password": "Password1$",
  "avatar_url": "https://example.com/avatar.png"
}
Response  201 Created 
{
  "user": {
    "id": "<uuid>",
    "email": "testuser@example.com",
    "avatar_url": "https://example.com/avatar.png"
  },
  "accessToken": "<access-token>"
}

A refreshToken cookie is also set on the response.

Error Responses
Status Condition Error message
400 A required field is missing or empty <field> cannot be empty
400 Email is not a valid format Invalid email address
400 Name is shorter than 3 characters Name must be at least 3 characters
400 Password does not meet complexity requirements Password must be at least 8 characters and contain at least: one uppercase letter, one lowercase letter, one digit and one special character
400 Avatar URL is not a fully qualified URL Avatar must be a valid url

POST /api/auth/login

Authenticates an existing user with their email and password. On success, returns the user's details and a fresh access token. A new refresh token cookie is also set.

Request Body
Field Type Required
email string Yes
password string Yes 
{
  "email": "testuser@example.com",
  "password": "Password1$"
}
Response  200 OK 
{
  "user": {
    "id": "<uuid>",
    "email": "testuser@example.com",
    "avatar_url": "https://example.com/avatar.png"
  },
  "accessToken": "<access-token>"
}

A refreshToken cookie is also set on the response.

Error Responses
Status Condition Error message
401 Email not found or password is incorrect Invalid email or password

POST /api/auth/refresh

Issues a new access token using a valid refresh token. The existing refresh token is read from the request cookie and, on success, replaced with a new one. Use this endpoint when the access token has expired, instead of requiring the user to log in again.

Request Cookie 
refreshToken=<valid-refresh-token>
Response  200 OK 
{
  "accessToken": "<new-access-token>"
}

A new refreshToken cookie replaces the old one.

Error Responses
Status Condition Error message
401 No refresh token cookie present No refresh token provided
401 Refresh token is expired or has been tampered with Refresh token expired or invalid. Please log in again.

User Profile

All profile endpoints require a valid access token in the Authorization header using the Bearer scheme.

Authorization Header Format 
Authorization: Bearer <access-token>
Common Authorization Errors (apply to all profile endpoints)
Status Condition Error message
401 No Authorization header sent No authorization header provided
401 Header present but token is missing Bearer Unrecognized authorization header
401 Token is structurally malformed Invalid access token
401 Token is valid but has expired Access token expired - use /api/auth/refresh to get a new one

PATCH /api/profile/password

Changes the authenticated user's password. The current password must be provided for verification. After a successful change, all existing sessions are invalidated - the user will need to log in again with the new password.

Request Body
Field Type Required
current_password string Yes
new_password string Yes - must meet password rules 
{
  "current_password": "Password1$",
  "new_password": "Password1$$"
}
Response  200 OK 
{
  "message": "Password changed successfully. Please log in again."
}
Error Responses
Status Condition Error message
400 The provided current password does not match Current password is incorrect
400 New password does not meet complexity requirements Password must be at least 8 characters and contain at least: one uppercase letter, one lowercase letter, one digit and one special character

Input Validation Rules

Password Requirements

All passwords (on registration and password change) must satisfy the following:

  • At least 8 characters long
  • At least one uppercase letter (A-Z)
  • At least one lowercase letter (a-z)
  • At least one digit (0-9)
  • At least one special character

General Field Rules

Field Rule
email Must be a valid email address format (e.g. user@example.com)
name Minimum 3 characters
avatar_url Must be a fully qualified URL (e.g. https://example.com/img.png)

Daily Check-ins

All daily check-in endpoints are protected. Each day (1–75) has exactly one record per user, enforced at the database level. Days must be created in order — a day cannot be created unless the previous day already exists.

Common Authorization Errors (apply to all daily check-in endpoints)
Status Condition Error message
401 No Authorization header sent No authorization header provided
401 Header present but token is missing Bearer Unrecognized authorization header
401 Token is structurally malformed Invalid access token
401 Token is valid but has expired Access token expired - use /api/auth/refresh to get a new one

Milestones

Milestones are awarded automatically when a user completes day 25, 50, or 75 with all tasks checked off. They cannot be created or deleted manually. This endpoint returns whatever badges have been earned so far.

GET /api/milestones

Returns all milestones earned by the authenticated user. Returns an empty array if none have been awarded yet.

Request Headers 
Authorization: Bearer <valid-access-token>
